In-panel and off-panel widgets

For more information on what a panel is, see Create a theme for more details.

Two types of widgets are supported. One is off-panel widget (such as Scalebar widget) and another is in-panel widget (such as Query widget). Before coding, make a decision about whether your widget needs a panel or not. There are some differences between them:

  • Off-panel widget is not allowed in a widget pool. It’s only available to the widget-on-screen in app configuration.
  • With off-panel widgets, the app container opens the widget when the app starts.
  • With in-panel widgets, the app container creates an action button identified by the widget icon. When the widget icon is clicked, the widget opens in the panel.

Each widget has initial state to indicate openAtStart property is set to true or false.

You can set the inPanel property in the widget’s manifest.json. For more details, see Deploy your widget.
