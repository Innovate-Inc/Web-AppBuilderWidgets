FAQs

How to start Web AppBuilder for ArcGIS (Developer Edition)?

Refer to the Get started.

Why do I get an "Invalid redirect_uri" error page after setting App ID?

This is because the URI of Web AppBuilder is not added or not set correctly in the Redirect URI list when you registered App ID for Web AppBuilder in the ArcGIS portal. To update the Redirect URI list, refer to View and update app registration info for details. Reload Web AppBuilder after the update.

How to start Web AppBuilder for ArcGIS (Developer Edition) in https websites with your CA-signed certificate?

Web AppBuilder uses a self-signed certificate in Nodejs to support https. If you want to start Web AppBuilder in your full https site, you can use your own certificate. To do so, replace two files in the server directory: cakey.pem and cacert.pem. Refer to http://nodejs.org/api/https.html and https://www.openssl.org/docs/HOWTO/certificates.txt for more information.

How to use the local help?

You can start the Web AppBuilder help in a browser with http://[yourmachinename]:[port]/webappbuilder/help after Node.js starts, or go to the docs folder for Web AppBuilder's installation by right-clicking the index.html file, then open it with FireFox or IE. If you choose the latter option, be aware that Chrome does not support this HTML format, while with IE10+, you need to enable JavaScript before continuing.

What should I do when I fail to log in my ArcGIS Online organization or Portal for ArcGIS?

You may fail to log in to your ArcGIS Online organization or Portal for ArcGIS due to one of the followings:

  • No trusted certification available. To work around it, log in your portal first to make the certificate trusted, then log in Web AppBuilder again.
  • Need proxy to access externally. To work around it, set the proxy for Web AppBuilder from node command node server.js [-proxy=value].

How can I update the App ID for Web AppBuilder for ArcGIS (Developer Edition)?

The App ID is set when you start up the Web AppBuilder for the first time. To update it, open signininfo.json file in the server folder, then edit the appID property specified in the portal.

Is SSL enabled access supported?

Yes. Web AppBuilder supports SSL enabled access to online organizations or Portal for ArcGIS. The https request is required when the URL to such organization or Portal for ArcGIS is specified.

Which portal securities are supported?

Portal with the following securities are supported by Web AppBuilder:

  • OAuth
  • SAML
  • IWA
  • PKI
  • LDAP

Can I use Web AppBuilder for ArcGIS (Developer Edition) with IE8 or IE9 to access Portal with web-tier authentication?

No. Web AppBuilder for ArcGIS (Developer Edition) does not support using IE8 or IE9 to access the portal with web-tier authentication. This is due to the limitation of IE8 and IE9 not supporting CORS.

Why do I fail to start Web AppBuilder when try to access a Portal with LDAP authentication?

When start Web AppBuilder by specifying URL to a portal with LDAP authentication, you may encounter the error "Unable to access [Portal_URL]. A Server with specified hostname could not be found". This probably is caused by the SSL support in node server not matching the SSL version of LDAP server. To work it around, stop node server, then start it with command lines.

Open a command window and navigate to the server directory of Web AppBuilder's installation.

On 64-bit Windows, type node_x64.exe server.js -sslClientVersion=SSLv3_method.

On 32-bit Windows, type node_x32.exe server.js -sslClientVersion=SSLv3_method.

On other platforms, type node server.js -sslClientVersion=SSLv3_method.

Is portal with disconnected environments supported?

Yes. You must set your own apiURL manually in env.js file. Refer to Change URL of ArcGIS API for JavaScript. By default, Web AppBuilder points to the ArcGIS API for JavaScript hosted by ArcGIS.com, such as http://js.arcgis.com/3.12.

Why my deployed app can't access cross domain resources?

You need to set the appropriate proxy in the app config.json file. Refer to Use proxy.

Do I have to register my downloaded app for deployment?

The app created by Web AppBuilder can be downloaded and hosted on your web server. Any app that is downloaded must be registered with ArcGIS Online or Portal for ArcGIS if the app references nonpublic items. Refer to Deploy app.

Which versions of the Portal for ArcGIS should I use?

Web AppBuilder supports Portal for ArcGIS 10.3 and up. If you have Portal for ArcGIS 10.2.2 or 10.2.1, it is recommended to upgrade to 10.3.

Can I use the Portal for ArcGIS 10.2.2 and 10.2.1?

Although Web AppBuilder should work with Portal for ArcGIS 10.2.1 and 10.2.2, you are at risk if you prefer to use them, as they are not supported.

Which versions of ArcGIS API for JavaScript should I use?

Web AppBuilder 1.0 supports ArcGIS API for JavaScript 3.12 .

Which versions of ArcGIS Server should I use?

As Portal for ArcGIS 10.3 supports older versions of ArcGIS Server, Web AppBuilder should work with ArcGIS Server 10.0 and up.
