Get started

Install

Download Web AppBuilder for ArcGIS (Developer Edition) ZIP file to your local drive and unzip it.

Since Web AppBuilder runs on top of Node.js, it is required to have Node.js installed on your computer. In this release, the download package contains Windows version of node.js. If you are on Windows platforms, you can go to the next step to start Web AppBuilder. If you are on the other platform, such as Mac or Linux, you should install Node.js specifically on your platform before the next step.

Start Web AppBuilder

  • On Windows, double-click the startup.bat file inside the unzipped folder. The batch file starts the Node.js server on Command Prompt window, then Web AppBuilder in your default browser with http://[yourmachinename]:3344/webappbuilder. If the browser is closed, you can always reopen Web AppBuilder in the same or different browsers as long as Node.js is running in the Command Prompt window.

  • On other platforms, open the Command Prompt window, navigate to theserver directory of Web AppBuiilder's installation, then type node server.js. Once the Node.js runs, start Web AppBuilder in the browser with http://[yourmachinename]:3344/webappbuilder. If the browser is closed, you can always reopen Web AppBuilder in the same or different browsers as long as Node.js is running in the Command Prompt window.

Web AppBuilder runs on top of Node.js with the default port of 3344. If you want to run Web AppBuilder with additional parameters, different ports or as a Window Service , follow the instructions below. Otherwise go to next step to specify your portal URL.

  • Run with additional parameters and different ports
    1. Open a command window and navigate to the server directory of Web AppBuilder's installation.
      • On 64-bit Windows, type node_x64.exe server.js [-port=value] [-proxy=value].
      • On 32-bit Windows, type node_x32.exe server.js [-port=value] [-proxy=value].
      • On other platforms, type node server.js [-port=value] [-proxy=value].
      Note:

      -port specifies the port you want Node.js to run on and -proxy specifies the proxy URL if needed.

    2. Start Web AppBuilder in the browser with the port number specified above at http://[yourmachinename]:[yourport]/webappbuilder.
      Tip:

      If you have problems starting Web AppBuilder, refer to FAQs for more information.

  • Run as a Windows service
    1. Install Node.js on your windows platform.
    2. Run Command Prompt as administrator.
    3. Navigate to the server directory of Web AppBuilder, then run npm install command.
    4. Run npm run-script install-windows-service command.
    5. Start ArcGISWebAppBuilder service from Services window.

    To remove ArcGISWebAppBuilder service, run npm run-script uninstall-windows-service command.

Specify your portal URL

Specify the URL for ArcGIS Online organization or Portal for ArcGIS that you want Web AppBuilder to access.

Provide App ID for Web AppBuilder

Since this is the first time you've used Web AppBuilder with the portal specified above, you might be asked to provide an App ID for Web AppBuilder to support OAuth2 authentication . To retrieve an App ID, manually register the Web AppBuilder in you portal and set correct redirect URIs. Refer to Add apps and Register your app for details. When you finish the registration, copy the App ID in the text box and click Continue.

Tip:

During Web AppBuilder registration, you may input http://[yourmachinename]:3344/webappbuilder as URL of the app item, http://[yourmachinename]and https://[yourmachinename] as Redirect URIs. Be aware that, URIs with http and https protocols are distinguished as two different URIs. It is strongly recommended adding both of them into the URI list to avoid potential redirect issues.For a specified portal, this process is required for just once. When you start the Web AppBuilder next time, you should be able to log in the portal directly.

Note:

An App ID is required by OAuth2 authentication. Refer to Authentication and OAuth 2. If you have problems logging on, refer to FAQs for more information.

Log in your portal

To use Web AppBuilder, you must have an ArcGIS Online organization account, a Portal for ArcGIS account, or a ArcGIS for Developers account. Log in with one of these accounts associated with the portal you specified above.
